### Task: 2.2 — Request flows

- **Status:** Done
- **Recommendation:** Auth → import → async job → poll → dashboard read path
- **Why:** Matches ADR-001 CSV/manual ingestion and rules engine needs
- **Alternatives:** Sync compute on import — rejected (timeout)
- **Risks:** Job failure UX — show error state on poll
- **Next action:** Implement routes in CP-7–9
- **Evidence / links:** Spec §4

### Task: 2.3 — Async metrics (ADR-002)

- **Status:** Done
- **Recommendation:** `metrics_jobs` table + BFF triggers Analytics via internal POST
- **Why:** Decouples heavy Python work from Vercel serverless limits
- **Alternatives:** Inngest day 1 — deferred to v1.1
- **Risks:** Fire-and-forget without retry — mitigated with 3 retries in spec
- **Next action:** CP-3 include `metrics_jobs` entity
- **Evidence / links:** ADR-002, spec §5

### Task: 2.4 — Environments

- **Status:** Done
- **Recommendation:** Local Docker Postgres; Vercel preview + prod; Railway for API
- **Why:** Standard SaaS layout, low cost
- **Alternatives:** Single VPS — more ops burden
- **Risks:** Preview DB branch sprawl — delete Neon branches after PR
- **Next action:** `.env.example` in CP-6
- **Evidence / links:** Spec §6

### Task: 2.5 — Cost estimate

- **Status:** Done
- **Recommendation:** Budget ~$5/mo dev, ~$15–45 at 100 MAU, ~$80–150 at 1k MAU
- **Why:** Informs pricing floor for subscription
- **Alternatives:** All-in Supabase — rejected in stack decision
- **Risks:** Railway usage spikes — set billing alerts
- **Next action:** Revisit after beta traffic (CP-13)
- **Evidence / links:** Spec §7

### Task: 2.6 — Security baseline

- **Status:** Done
- **Recommendation:** App-level `user_id` scoping; Clerk on BFF; service secret for Analytics; Stripe webhook verify
- **Why:** Fastest correct MVP; RLS as v1.1
- **Alternatives:** Public Analytics API — rejected
- **Risks:** ORM leak without discipline — code review checklist in CP-6
- **Next action:** CP-3 FK + index design
- **Evidence / links:** Spec §8

### Task: 2.7 — Stack sign-off

- **Status:** Done
- **Recommendation:** Next.js 14 + FastAPI + Neon + Clerk + Stripe + Vercel + Railway
- **Why:** PRD alignment + CP-1 ingestion model
- **Alternatives:** Documented in spec §1
- **Risks:** Two-language repo — monorepo + Docker Compose
- **Next action:** CP-3 data model
- **Evidence / links:** `02-architecture.md` §1, §3
