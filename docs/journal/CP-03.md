## Task log

### Task: CP-3 — Schema stance

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** Normalize rounds/holes; shots optional; `raw_payload` JSONB on rounds; UUID PKs; `metrics_jobs` per ADR-002
- **Why:** MVP metrics need hole-level stats without shot tracking; preserves Phase 2 upstream data
- **Alternatives:** Mongo/document — rejected for relational queries; holes embedded in rounds JSONB — rejected for indexing
- **Risks:** Shot table unused early — gate shot UI behind data quality
- **Next action:** CP-4 metric_key catalog and field requirements
- **Evidence / links:** `docs/specs/03-data-model.md`, `db/migrations/001_initial.sql`

### Task: 3.1 — ERD

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** Mermaid `erDiagram` with 10 entities and cardinality notes
- **Why:** Matches CP-2 diagram style; diffable in git
- **Alternatives:** PNG-only ERD — harder to maintain
- **Risks:** Mermaid ER syntax limits — kept entity blocks concise
- **Next action:** None
- **Evidence / links:** `docs/diagrams/erd.mmd`

### Task: 3.2 — PKs and foreign keys

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** UUID `gen_random_uuid()`; CASCADE on user-owned children; SET NULL on `import_batch_id` and `source_insight_id`
- **Why:** Neon/Postgres standard; safe multi-service IDs
- **Alternatives:** Serial integers — leak ordering; ULID in DB — defer to CP-6 app layer
- **Risks:** UUID index size — acceptable at MVP scale
- **Next action:** CP-6 migration apply
- **Evidence / links:** SQL migration

### Task: 3.3 — Indexes

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** `(user_id, played_at)`, `(user_id, metric_key, as_of_date)`, job status partial
- **Why:** Matches dashboard list, trends, and job poll paths from architecture spec
- **Alternatives:** Over-index every FK — unnecessary for MVP row counts
- **Risks:** Missing composite for practice plans — mitigated with `(user_id, status, created_at)`
- **Next action:** EXPLAIN in CP-9 if slow
- **Evidence / links:** Spec §7

### Task: 3.4 — Multi-tenancy

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** Application-level `user_id` filter; denormalize `user_id` on `holes` and `shots`
- **Why:** Matches ADR-007 MVP; enables hole queries without round join for RLS later
- **Alternatives:** RLS-only — deferred v1.1
- **Risks:** Denormalized `user_id` drift — BFF sets from parent on insert
- **Next action:** `getCurrentUserId()` in CP-7
- **Evidence / links:** Spec §5, architecture §8.1

### Task: 3.5 — import_batches

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** Batch row with `source_format`, `status`, `rows_total`, `rounds_created`, `error_message`
- **Why:** ADR-001 audit trail; supports import history UI in CP-7
- **Alternatives:** Log only in object storage — no SQL audit for support
- **Risks:** Orphan batches on crash — status `failed` + error text
- **Next action:** CP-7 import endpoint sets batch then rounds
- **Evidence / links:** SQL `import_batches` table

### Task: 3.6 — SQL migration

- **Status:** Done

- **Recommendation:** Single `001_initial.sql` with enums, tables, indexes, `updated_at` triggers
- **Why:** ORM-agnostic draft; CP-6 picks runner (ADR-004)

- **Risks:** Trigger syntax PG version — tested against PG 16 target
- **Next action:** Docker Postgres apply in CP-6
- **Evidence / links:** `db/migrations/001_initial.sql`

### Task: 3.7 — CSV column mapping

- **Status:** Done
- **Date completed:** 2026-05-22
- **Recommendation:** One row per hole; `round_id` groups to `rounds.external_round_id`; dedup unique per user
- **Why:** Aligns to `birdieiq-round-import-v1.example.csv` and CP-1 P0 fields
- **Alternatives:** Separate round summary sheet — defer; single sheet simpler for users
- **Risks:** Re-import same `round_id` — MVP rejects duplicate (upsert v1.1)
- **Next action:** CP-7 validation implementation
- **Evidence / links:** Spec §8, example CSV

## Decisions made

- UUID PKs with `gen_random_uuid()` for MVP migration draft
- `fairway_hit` as `Y` / `N` / `NA` text (matches CSV semantics)
- `metrics_jobs` scoped to `user_id` (full history recompute per ADR-002)
- ADR-004 unchanged — Drizzle vs Prisma at CP-6 only
